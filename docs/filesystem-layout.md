# JSOS Filesystem Layout
**Item 886** — The `/` directory tree

## Root Filesystem (`/`)

```
/
├── boot/
│   ├── bundle.js        — The compiled TypeScript OS bundle (entire OS)
│   ├── bundle.crc       — CRC32 of bundle.js (integrity check)
│   └── grub/
│       ├── grub.cfg     — GRUB2 boot menu
│       └── core.img     — GRUB2 core image
│
├── etc/
│   ├── hostname         — System hostname
│   ├── timezone         — e.g. "America/New_York"
│   ├── locale.conf      — LANG=en_US.UTF-8
│   ├── vconsole.conf    — KEYMAP=us
│   ├── passwd           — User database (username:uid:gid:home:shell)
│   ├── shadow           — Hashed passwords (root readable only)
│   ├── group            — Group database
│   ├── ssl/
│   │   └── cacerts.pem  — Mozilla Root CA bundle
│   ├── hosts            — /etc/hosts static DNS entries
│   └── resolv.conf      — Nameserver config (nameserver 1.1.1.1)
│
├── bin/                 — System utilities (TypeScript scripts)
│   ├── sh               — JSOS shell (invokes REPL)
│   ├── ls               — File listing
│   ├── cat              — File concatenation
│   ├── cp               — File copy
│   ├── mv               — File move
│   ├── rm               — File remove
│   ├── mkdir            — Directory creation
│   ├── chmod            — Permission change
│   ├── ps               — Process list
│   ├── kill             — Send signal
│   ├── ping             — ICMP ping
│   ├── curl             — HTTP client
│   └── ...
│
├── usr/
│   ├── lib/             — Shared TypeScript libraries
│   │   ├── libjsos.js   — Core OS runtime library
│   │   └── fonts/       — Bitmap and TTF fonts
│   ├── share/
│   │   ├── zoneinfo/    — Timezone data (IANA tz database, binary)
│   │   └── ca-certs/    — Certificate authorities
│   └── local/
│       └── apps/        — User-installed TypeScript applications
│
├── home/
│   └── <username>/      — User home directories
│       ├── .profile     — Shell startup script
│       ├── .history     — REPL history
│       └── Desktop/     — GUI desktop files
│
├── var/
│   ├── log/
│   │   ├── kernel.log   — Kernel and OS messages (COM1 mirror)
│   │   └── browser.log  — Browser console output
│   ├── tmp/             — Temporary files (cleared on boot)
│   └── lib/             — Persistent application state
│
├── proc/                — Virtual filesystem: process info
│   ├── <pid>/
│   │   ├── status       — Process status (name, ppid, state, memory)
│   │   ├── maps         — Virtual memory map
│   │   └── fd/          — Open file descriptors
│   ├── meminfo          — Memory usage
│   ├── cpuinfo          — CPU info (from CPUID)
│   └── uptime           — Boot time in seconds
│
├── dev/                 — Device files
│   ├── null             — Discard device
│   ├── zero             — Zero device
│   ├── random           — RDRAND-backed RNG
│   ├── urandom          — Same as random in JSOS
│   ├── tty0             — First virtual terminal
│   ├── sda              — First ATA/SATA disk
│   ├── sda1             — First partition
│   └── net/
│       └── eth0         — First network interface
│
└── sys/                 — Virtual filesystem: hardware sysfs
    ├── class/net/eth0/  — Network interface attributes
    └── block/sda/       — Block device attributes
```

## Mount Points

| Path    | Filesystem | Device  | Notes |
|---------|-----------|---------|-------|
| `/`     | ext4      | `/dev/sda2` | Root partition |
| `/boot` | ext4      | `/dev/sda1` | Boot partition (if separate) |
| `/proc` | procfs    | none    | Virtual — process info |
| `/dev`  | devfs     | none    | Virtual — device files |
| `/sys`  | sysfs     | none    | Virtual — hardware sysfs |
| `/tmp`  | tmpfs     | RAM     | Cleared on boot |

## File Permissions

JSOS uses standard Unix permission bits (rwxrwxrwx).
- Root: uid=0, gid=0
- Default user: uid=1000, gid=1000

## Special Files

- `/dev/random` — backed by `kernel.rdrand()` (RDRAND instruction)
- `/proc/meminfo` — dynamically generated by the VMM
- `/proc/<pid>/status` — generated by the process scheduler
